# AI Pricing Tracker

**🔗 Live site: [ai-pricing-tracker-mu.vercel.app](https://ai-pricing-tracker-mu.vercel.app)**

A dashboard that tracks and compares pricing for major AI chatbots — both consumer subscription plans and developer API token pricing — in one place.

---

## What it tracks

### 💳 Subscription Plans
Consumer, team, and enterprise plans for 9 providers (will be updated with more soon!):
**ChatGPT · Claude · Gemini · Grok · Perplexity · Mistral · Copilot · DeepSeek · Meta AI**

- Flat comparison table with columns: Provider, Plan, Price/mo, Tier, Model Access, Usage Limits, Features
- Filter by tier (Consumer / Team / Enterprise) and by provider
- Click any provider name to open a side drawer with full plan details, model access info, and price history timeline

### 🔌 API Token Pricing
Developer pricing ($/1M tokens) for models from OpenAI, Anthropic, Google, Meta, DeepSeek, Mistral, xAI, Perplexity and more.

- Input, Output, and Blended price columns
- Speed (tokens/sec) and Intelligence score bars (powered by Artificial Analysis)
- Search + provider filter + sortable columns
- Built-in cost calculator: enter your monthly token usage to estimate spend per model

### 📈 Price History
Automatically tracks API price changes over time and logs them with percentage diff. Significant changes also open a GitHub Issue as an alert.

## Data sources

| Data | Source | Update frequency |
|---|---|---|
| API token pricing | [Artificial Analysis API](https://artificialanalysis.ai) (primary) / [OpenRouter](https://openrouter.ai) (fallback) | Weekly (every Monday) via GitHub Actions |
| Subscription plans | `data/subscriptions.yml` — manually maintained in this repo | On PR merge |
| Price change history | Auto-generated by fetch script | Weekly |

## Tech stack

- **Next.js 16** — static export (`output: "export"`)
- **TypeScript** + **Zod** — runtime schema validation for all data files
- **Tailwind CSS v4**
- **Lucide React** — icons
- **Python 3.12** — data fetch scripts
- **GitHub Actions** — weekly automated price fetch, PR data validation
- **Vercel** — hosting

## Updating subscription prices

Subscription plan prices are maintained manually in `data/subscriptions.yml`. When a provider changes their pricing:

1. Edit `data/subscriptions.yml`
2. Add an entry to the provider's `history[]` array with the date and description of the change
3. Open a PR — the `validate.yml` workflow will check the schema automatically
